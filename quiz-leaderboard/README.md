# Quiz Leaderboard System (Java + Spring Boot)

This project solves the internship assignment flow by polling the validator API 10 times, removing duplicate quiz events, computing participant totals, and submitting the final leaderboard once.

## What this application does

1. Calls validator GET API exactly 10 times using `poll=0` to `poll=9`.
2. Waits 5 seconds between each poll request.
3. Reads wrapped response and extracts event objects with:
   - `roundId`
   - `participant`
   - `score`
4. Removes duplicates using a unique event key: `roundId + participant`.
5. Aggregates scores per participant from only unique events.
6. Sorts leaderboard descending by `totalScore`.
7. Computes overall total score (sum of all participant totals).
8. Submits final result to a POST endpoint once.

## Project structure

- `QuizLeaderboardApplication` - Spring Boot entry point
- `PollingOrchestrator` - polling workflow and timing control
- `ExternalQuizApiClient` - external GET/POST API communication
- `LeaderboardService` - deduplication and score aggregation logic
- `model/*` - request/response domain models
- `config/*` - typed config + `RestTemplate` bean

## Design choices

- **Spring Boot** for backend-style structure and clean dependency management.
- **HashSet** (`seenKeys`) for O(1) duplicate checks.
- **HashMap** (`participant -> totalScore`) for O(1) aggregation updates.
- **CommandLineRunner** to run end-to-end processing on application startup.
- **Error handling**:
  - GET polling failures are logged and skipped (process continues).
  - POST submission failure throws an error after logging.

## API assumptions

### Poll endpoint (GET)

`GET {QUIZ_BASE_URL}/quiz/messages?regNo={QUIZ_REG_NO}&poll={0..9}`

Expected response shape:

```json
{
  "regNo": "2024CS101",
  "setId": "SET_1",
  "pollIndex": 0,
  "events": [
    { "roundId": "R1", "participant": "Alice", "score": 10 },
    { "roundId": "R1", "participant": "Bob", "score": 20 }
  ]
}
```

### Submit endpoint (POST)

`POST {QUIZ_BASE_URL}/quiz/submit`

Payload generated by this app:

```json
{
  "regNo": "2024CS101",
  "leaderboard": [
    { "participant": "Bob", "totalScore": 25 },
    { "participant": "Alice", "totalScore": 22 },
    { "participant": "Carol", "totalScore": 15 }
  ]
}
```

## Configuration

Set environment variables for real endpoints:

- `QUIZ_BASE_URL` (default: `https://devapigw.vidalhealthtpa.com/srm-quiz-task`)
- `QUIZ_REG_NO` (example: `2024CS101`)

API paths are configured in `application.yml` as:
- `messages-path: /quiz/messages`
- `submit-path: /quiz/submit`

## Run locally

```bash
mvn spring-boot:run
```

Or build and run jar:

```bash
mvn clean package
java -jar target/quiz-leaderboard-1.0.0.jar
```
